## [CodeRescue: Budget-Calibrated Recovery Routing for Coding Agents](https://arxiv.org/abs/2607.19338v1)
**Score:** 0.910  |  **arXiv:** 2607.19338v1

This paper introduces CodeRescue, a recovery framework for coding agents that dynamically decides whether a failed attempt should retry using a cheap model with execution feedback or escalate to a stronger, more expensive model. By combining a supervised router with a Conformal Risk Control layer, the system adapts its recovery decisions to varying cost budgets at runtime without requiring retraining. Across multiple coding benchmarks, CodeRescue matches or exceeds the solve rate of an "always-escalate" strategy while reducing recovery costs by up to 65%.

## [When Do Multi-Agent Systems Help? An Information Bottleneck Perspective](https://arxiv.org/abs/2607.16133v1)
**Score:** 0.909  |  **arXiv:** 2607.16133v1

This paper introduces an information bottleneck framework to explain when and why multi-agent systems (MAS) outperform single-agent systems (SAS) in LLM tasks. The authors demonstrate that MAS limits context size through compressed relay messages, creating a fundamental trade-off where reducing context redundancy improves efficiency but risks losing task-relevant information. Through 18 controlled experiments, they show that MAS benefits weaker models or tasks where relay messages are near-sufficient, whereas stronger models often perform better in a single-agent setup that preserves the full, uncompressed context.

## [Selective State-Space Adaptation and Retrieval for Language Model Reasoning](https://arxiv.org/abs/2607.19326v1)
**Score:** 0.906  |  **arXiv:** 2607.19326v1

To overcome standard LoRA's static updates, this paper introduces state-space adapters that dynamically adjust parameter updates at both the token and context levels using Mamba recurrence. Specifically, **MaLoRA** provides input-dependent dynamic scaling across tokens, while **MaRA** tracks cross-segment state to retrieve relevant context before generation. Evaluated across Qwen-2.5, Llama-3.1, and Gemma-2 backbones, the approach consistently outperforms standard LoRA on multi-hop reasoning and long-context QA benchmarks, yielding an average improvement of +6.8 F1.

## [Copy Less, Ground More: Overcoming Repetitive Copying in Long-Context Reasoning via Evidence-Aware Reinforcement Learning](https://arxiv.org/abs/2607.19345v1)
**Score:** 0.905  |  **arXiv:** 2607.19345v1

The authors identify "repetitive copying"—where LLMs slavishly copy text from the input into their reasoning traces instead of solving the task—as a major failure mode in long-context reasoning driven by poor evidence grounding. To fix this, they introduce GEAR, a reinforcement learning method that uses automated data annotation to reward models for grounding their reasoning in key task-relevant evidence while penalizing overlap with irrelevant distractors. Across multiple benchmarks and context lengths, GEAR consistently reduces unhelpful copying and reasoning length while improving accuracy by up to 4.6 points over standard RL.
